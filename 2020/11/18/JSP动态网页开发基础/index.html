JSP动态网页开发基础

JSP
发布日期:   2020-11-18

为什么学习B/S技术?

客户端 服务器 C（Client）/S（Server）有局限性，必须安装客户端
浏览器 服务器 B（Browser）/S（Server）无需安装客户端软件，也无需更新

B/S与C/S的比较：

B/S架构 C/S架构
软件安装 浏览器 在响应速度和安全性上需要花费更多设计成本
升级维护 客户端零维护 客户端需要单独维护和升级
平台相关 与操作系统平台的关系最小化 对客户端操作系统一般有限制
性能安全 在响应速度和安全性上需要花费更多设计成本 能充分发挥客户端处理能力，客户端响应快

什么是B/S技术

B/S架构(浏览器/服务器):程序完全部署在服务器上

B/S技术的工作原理

B/S架构采用 请求/响应 模式进行交互

URL(Uniform Resource Locator统一资源定位符)

URL的组成

例：http :// localhost:8888 / news/index.html
http: 协议部分
localhost:8888 主机IP地址:端口号
news/index.html 项目资源地址

服务器

图片服务器
文件服务器
数据库服务器
缓存服务器
应用服务器
Java Web —> Web服务器

常用的Web服务器

Microsoft：IIS
Apache： Tomcat （Tomcat是开源的，小型的免费的）
Oracle：WebLogic
IBM：WebSphere （前两个商用的，安全级别更高）
Nginx （稳定性高，占用内存小，并发能力强）

Tomcat服务器

Apache Jakarta的开源项目
轻量级应用服务器
开源,稳定,资源占用小
JSP/Servlet容器

Tomcat目录结构

目录 说明
/bin 存放各种平台下用于启动和停止Tomcat的脚本文件
/conf 存放Tomcat服务器的各种配置文件
/lib 存放Tomcat服务器所需的各种JAR文件
/logs 存放Tomcat的日志文件
/temp Tomcat运行时用于存放临时文件
/webapps 当发布Web应用时，默认情况下会将Web应用的文件存放于此目录中
/work Tomcat把由JSP生成的Servlet放于此目录下

操作Tomcat注意事项：

startup.bat 启动服务器
shutdown.bat 关闭服务器

开启服务器后关闭服务器建议不要直接点×关闭，应该点击 shutdown.bat关闭.防止出现啥乱七八糟的麻烦

端口号修改：conf/server.xml

JSP（Java Server Pages）概述:

  1. 运行在服务端的Java页面
  2. 使用HTML嵌套Java代码实现

工作原理

JSP中的page指令：

通过设置内部的多个属性定义整个页面的属性
语法:
<%@ page 属性1=”属性值” 属性2=”属性值1,属性值2”…属性n=”属性值n”%>

常用属性:

属性 描述 默认值
language 指定JSP页面使用的脚本语言 Java
import 通过该属性来引用脚本语言中使用到类文件 无
contentType 用来指定JSP页面所采用的编码格式 text/html,ISO-8859-1

out对象的方法:

属性 说明
print 向页面输出显示
println 向页面输出显示,在结尾出加换行

  3. 在JSP脚本中注释：<%//单行注释%>；<%/*多行注释*/%>

网页输出文本

1.用Java语言输出文本

<% out.print(“

Hello Wrold

“);%>

2.直接用HTML输出文本

Hello Wrold

3. 使用<% out,print(); %>或<% out.printIn(); %>实现显示页面全部内容，这样做的弊端：

①JAVA代码与HTML标签混在一起，可读性不好，不利于修改
②显示数据来自于数据库，数据库里的数据不能写死，要用变量来读取写在数据库的内容
③因为不方便所以这种方法用的比较少

4.用变量的方法输出文本:

<% String title = “Hello World”; %>

<%=title%>
好处：直接在数据库里读的时候直接调用后台方法。
注：变量声明的时候要在最后加一个分号（；），在调用的时候前面要写（<%=），但是最后不能写分号（；）

Web容器处理JSP文件请求:

第一次处理需要经过3个阶段

  1. 翻译阶段 （JSP文件翻译成Java文件）
  2. 编译阶段 （编译Java文件为Class文件）
  3. 执行阶段 （执行Class文件）

403 权限不足（访问a目录，但是a目录设置不可见）
页面无法显示

排除错误：
